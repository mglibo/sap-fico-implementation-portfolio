Project Background & Business Case

Current State (AS-IS)
  • Legacy ERP system (15+ years old) with limited reporting capabilities
  • Fragmented financial data across 3 subsidiaries with manual consolidation
  • No real-time cost controlling or profitability analysis
  • Manual bank reconciliation processes consuming 40+ hours/month
  • Separate chart of accounts per subsidiary — no standardization
Future State (TO-BE)
  • SAP S/4HANA with unified Chart of Accounts across all entities
  • Real-time financial reporting with SAP Fiori dashboards
  • Automated bank statement processing and reconciliation
  • Integrated cost controlling with drill-down profitability analysis
  • Centralized shared services for AP/AR processing

Cutover Plan

Step-by-step go-live transition activities.

T-4 Weeks: Preparation

  • Final data migration rehearsal (3rd mock cutover)
  • End-user training completion sign-off
  • Production system access provisioning
  • Cutover runbook finalization and walk-through

T-2 Weeks: Final Readiness

  • Go/No-Go decision meeting with steering committee
  • Legacy system freeze for data extraction
  • Communication plan execution (stakeholder emails, training reminders)
  • Hypercare team roster and escalation paths confirmed

Go-Live Weekend (T-0)

  • Day 1 (Friday PM): Legacy system cutoff, final data extraction
  • Day 1 (Friday PM): Load master data — CoA, vendors, customers, assets, cost centers
  • Day 2 (Saturday): Load open item balances — AP, AR, G/L, asset balances
  • Day 2 (Saturday): Reconciliation — verify all balances match legacy
  • Day 3 (Sunday): Smoke testing — key transactions in each module
  • Day 3 (Sunday): Go-Live sign-off → System open for productive use

T+1 to T+4 Weeks: Hypercare

  • On-site support team at all 3 locations
  • Daily status calls and issue triage
  • Priority 1 issues: 2-hour response SLA
  • First month-end close support with parallel legacy comparison
  • Knowledge transfer to internal support team
